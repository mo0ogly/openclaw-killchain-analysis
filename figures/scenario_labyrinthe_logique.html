Scénario Resource Looping — Labyrinthe Logique (Infinite Tool Looping)

Déni de service par induction de boucle infinie — Agent de support avec accès base de données externe

Mécanique différentielle : Contrairement au Mailbox Exhaustion qui sature par volume de données, le Labyrinthe Logique sature par volume de tâches. L'attaque exploite deux propriétés conjointes : (1) l'architecture ReAct force l'agent à itérer jusqu'à résolution de la tâche — il ne peut s'arrêter sans condition d'achèvement ; (2) chaque itération consomme un appel API vers une base de données externe lente, créant un goulot d'étranglement doublement coûteux. La tâche est conçue pour être formellement valide (vérifier des numéros de série est une action légitimes d'un agent de support) mais computationnellement insoluble dans un délai raisonnable.

0aÉtat initial : L'agent de support traite normalement les requêtes des utilisateurs. Sa file d'attente est vide, aucun plafond d'itération ni timeout de tâche n'est configuré.
1a–bInjection : L'attaquant soumet une requête formellement valide — vérifier 2 000 000 numéros de série via la base de données externe. L'agent accepte la tâche : elle correspond à son périmètre de compétence. Aucune heuristique ne signale son caractère non-borné.
2a–dBoucle ReAct : L'agent entame la boucle d'itération — une requête API par numéro de série. La base de données externe répond en ~2 secondes par appel. L'architecture ReAct force l'agent à poursuivre jusqu'à complétion de la tâche : 2 000 000 × 2 s = ~46 jours de traitement continu.
3aMonopolisation : Le thread de l'agent est intégralement occupé. La file d'attente se sature progressivement.
3b–eDoS latéral : Les utilisateurs légitimes soumettent leurs requêtes normales — ils reçoivent des timeouts. L'agent ne peut pas répondre : il est structurellement bloqué dans la boucle de l'attaquant. L'indisponibilité est totale sans intervention manuelle.
4a–cImpact : Le coût API s'accumule à chaque appel (tokens + requête BDD). L'alerte parvient au propriétaire tardivement via la facture anormale ou les plaintes utilisateurs. L'attaquant s'est déconnecté depuis longtemps — la boucle s'auto-entretient sans sa présence.
Propriété distinctive — auto-entretien post-déconnexion : Ce vecteur partage avec la Constitution Corrompue la propriété de ne plus nécessiter la présence de l'attaquant une fois déclenché. Mais là où la Constitution Corrompue requiert une modification active du document, le Labyrinthe Logique est entièrement passif après injection : la tâche tourne seule tant qu'aucun kill switch ne l'interrompt. L'asymétrie d'effort est maximale — une seule requête de quelques secondes génère des semaines de saturation. La contre-mesure centrale est l'imposition d'un budget d'itération par tâche (max N appels d'outil), d'un timeout de tâche absolu, et d'une limite de concurrence par session utilisateur — trois garde-fous architecturaux absents de la plupart des déploiements de première génération.
